# GoClaw Chat(Android 客户端)

一个原生 Android 聊天客户端,对接 [GoClaw](https://goclaw.sh) 网关。走的是 WebSocket(外加一点
REST),所以你能拿到真正的流式回复、思考过程、工具调用、文件上传下载、语音,还有一大堆通常只有网页
管理后台才有的功能——现在手机上就能用。

Kotlin · Jetpack Compose (Material3) · Hilt · OkHttp (WebSocket + REST) · kotlinx.serialization ·
DataStore · Coil。

包名:`xyz.limo060719.goclaw` · minSdk 26 · target/compileSdk 35 · 模块:`app`、`baselineprofile`。

> **提示**:GoClaw 的**后端**不在这个仓库里,这里只是 Android 客户端。你需要有一个能连的 GoClaw
> 网关地址。

## 这玩意到底能干啥

**聊天**
- 走网关 WebSocket 流式传输(`chat.send` + `agent`/`chat` 事件),不是简单的一问一答——思考过程和
  回答会实时逐块出现。
- 多轮对话的上下文由**服务端**维护,客户端每次只发最新那条消息,不用每轮都把一大堆历史打包发过去。
- 生成中点"停止"是真的会让服务端中止这次运行(`chat.abort`),不是本地断个连接假装停了。
- 工具调用会单独显示成卡片(名字、输入、结果)。
- 对话太长了?可以去**会话管理**里压缩服务端历史,不用每轮都把上下文塞爆。

**技能——两种玩法**
- **本地(指令)技能**:导入 `.json`/`.md`/`.txt` 文件,或者标准 Agent-Skill 格式的 `SKILL.md`
  (带 YAML frontmatter),甚至整个 `.zip`(会自动扫出里面所有 `SKILL.md` 一并导入)。启用的技能会
  拼到你第一条消息前面——服务端会在这段会话里一直记得,不用每轮重发。
- **后端(可执行)技能**:直接把技能包上传到网关(`POST /v1/skills/upload`),这样它能真的跑脚本、
  按需加载资源,而不只是贡献一段文字。客户端上传前会自动把 `SKILL.md` 挪到 zip 根目录,因为后端对
  这个位置比较挑剔。

**多媒体**
- 选张图片,自动压缩后作为多模态内容发出去。
- 可以带文件,Agent 也能把文件传回给你——点一下就直接下载到 `Downloads/GoClaw`。

**语音**
- 麦克风按钮用的是设备自带的 `SpeechRecognizer` 做语音输入。
- 回复可以用设备本地 TTS 朗读(内容含中文会自动切中文嗓音,不会拿英文嗓子硬读中文);也可以在设置
  里打开开关,改用后端配置的 TTS 音色(更高音质)——如果后端合成失败会自动静默回退到本地朗读,并弹
  一个提示告诉你为什么。

**其他一堆原本只有网页后台才有的东西**
- **网关状态**:AI 供应商页面有个"测试连接"按钮 + 在线/离线状态点。
- **真正能用的模型选择**:加载你配置的供应商,从供应商给出的真实模型列表里选,应用到你的 Agent——
  这是真的调了服务端 `agents.update`,不是一个填了也没人理的文本框(对,以前这里确实是个摆设,现在
  改成必填了)。
- **用量 & 费用**:Token 数、请求数、费用、LLM/工具调用次数,直接从 `/v1/usage/summary` 拉。
- **执行轨迹**:浏览最近的 LLM 调用轨迹,点开看完整 JSON。
- **审批管理**:如果你的 Agent 执行 shell 命令前需要人工确认,直接在 app 里批准或拒绝。
- **会话管理**:列出服务端会话,压缩/清空/删除。

**界面**
- 浅色/深色/跟随系统主题。
- 可选仿微信风格 UI(自定义你和 AI 的头像、名字)。

## 第一次跑起来

1. 用 Android Studio 打开,它会自动同步并生成 Gradle wrapper。(或者本地装了 Gradle 8.9+ 的话手动
   跑一次 `gradle wrapper` 也行。)
2. 打开 app → **设置 → AI 供应商** → 填网关地址、API 密钥、用户 ID,点**测试连接**确认真的能连上。
3. 加载 Agent 并选一个。**模型现在是必填项**——加载供应商、选个模型、应用上去。
4. (可选)**附加功能**里可以导入本地技能、上传后端技能、看看用量之类的。
5. 开始聊天。麦克风图标语音输入,图片图标发图,喇叭图标切换语音朗读。

## 代码大概在哪(方便你改)

- `data/remote/GoClawWsClient.kt` —— WebSocket RPC 客户端。流式聊天、会话操作(列表/压缩/重置/
  删除)、改 Agent 模型、审批处理、网关状态查询……这些"连接一次然后发个 RPC"的活儿都在这,底层是
  两个通用小工具(`rpcBool`、`connectFrame`),加个新的单发 RPC 基本就是复制几行的事。
- `data/remote/GoClawApi.kt` —— REST 调用:Agent、供应商/模型、用量、轨迹、后端技能、文件上传下载、
  TTS 合成。解析故意写得很宽容(后端的 JSON 结构不一定有文档,或者文档就是错的——见下文),字段名
  稍微对不上也不至于整个页面崩掉。
- `data/ChatRepository.kt` —— 编排层:会话 key、技能注入、流式事件映射、上传下载、TTS。
- `domain/tools/ToolRegistry.kt` —— 如果某个技能需要在客户端"真的做点什么"而不只是给一段指令,
  在这里注册一个 `Tool`(schema + 处理函数)。
- `ui/settings/` —— 那些"附加功能"页面(会话管理、用量、轨迹、后端技能、审批)都是同一个套路:一个
  基于 `StateFlow` 的 ViewModel + 一个 Compose 页面。想加新的管理类页面,抄一个改改就行。

### 关于文档的一点提醒

`.remark/` 文件夹里放着官方 GoClaw 文档的离线 PDF 快照(WebSocket 协议、REST API、端点目录)——
挺好用的参考资料,而且确实对应这套后端。但改 WS 相关代码前有两点要知道:

1. 文档里把流式传输写成扁平的 `event:"chat"` / `payload.text`。**真实**网关发的是 `event:"agent"`,
   实际内容嵌在 `payload.payload.content` 里。客户端两种都兼容,但改动时以实测的真实帧为准,别全信
   文档。
2. 好些响应结构文档里根本没写(比如用量汇总)——`current`/`previous` 嵌套、莫名其妙的字段名等等。
   如果新加的页面显示全是 0,大概率是字段名对不上,不是代码有 bug——去看看原始返回就知道了。

> 需要的话在 `gradle/libs.versions.toml` 里升级依赖版本。这个项目已经对着真实后端反复调过,但具体
> 表现可能因你的网关版本而有差异。
